The CPU Heartbeat: Fetch-Decode-Execute

1. Fetch

The CPU asks RAM for the next instruction. It receives the binary number (like 00010101...).

2. Decode

The Control Unit splits the bits. It looks at the Opcode first.
"Ah, 0001 means ADD."

3. Execute

Electricity flows through the specific gates for that operation. The math happens, and the result is stored.

Why Binary?

Using fixed-size instructions (like 32-bit or 64-bit) allows the hardware to be extremely fast. The processor knows exactly where to look for the Opcode (the first few bits) every single time, without needing to "read" the whole sentence first.
